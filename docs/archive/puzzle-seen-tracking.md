- [x] Task 2 — `game/puzzle_seen_tracker.lua` (new file) — Create a new module, structured like `game/puzzle_catalog.lua` (a plain table returned from the file, module-level state, no disk I/O, lives only for the process's lifetime). It holds `seen = {easy = {}, med = {}, hard = {}}` (a set of already-shown path strings per tier, keyed by path with value `true`) and exposes:
  - `PuzzleSeenTracker.mark_seen(tier, path)` — sets `seen[tier][path] = true`.
  - `PuzzleSeenTracker.is_seen(tier, path)` — returns `true`/`false` for whether `path` has been marked seen under `tier`.
  - `PuzzleSeenTracker.unseen_paths(tier, all_paths_for_tier)` — given the tier's full path array (e.g. one value from `PuzzleCatalog.list_by_tier()`), returns a **new** array containing only the entries not yet marked seen for that tier (does not mutate the input array).
  - `PuzzleSeenTracker.is_tier_exhausted(tier, all_paths_for_tier)` — returns `true` iff `#PuzzleSeenTracker.unseen_paths(tier, all_paths_for_tier) == 0` (an empty `all_paths_for_tier` is vacuously exhausted).
  - `PuzzleSeenTracker.reset()` — clears `seen` back to `{easy = {}, med = {}, hard = {}}`. Nothing in game-runtime code calls this (per the design doc's resolved scope there is no in-game reset trigger — restarting the process already clears everything since this is a module-level table); it exists so `tests/test_jigsaw.lua` can isolate test scenarios that would otherwise share this process-lifetime cache across unrelated `do...end` blocks in the same test run.
