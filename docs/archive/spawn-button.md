# Spawn Button Checklist

- [x] Task A ‚Äî `game/spawn_button.lua` (new) ‚Äî Create a `SpawnButton` module modeled on `game/jigsaw_box.lua`'s shape: `SpawnButton.new(x, y, on_press)` builds `self.sprite = Sprite.new(x, y, C.SLOT, C.SLOT)` with a distinct color (e.g. `{0.9, 0.2, 0.2, 1}`), stores `self.on_press = on_press`. `SpawnButton:interact()` calls `self.on_press()` if set (no internal state machine ‚Äî just a trigger). `SpawnButton:centre()` returns `{x = self.sprite.x + C.U, y = self.sprite.y + C.U}` (matches `JigsawBox:centre()`). `SpawnButton:draw()` calls `self.sprite:draw()`.

- [x] Task B ‚Äî `game/scenes/game_scene.lua` ‚Äî In `GameScene:on_enter()`: add `local WORLD_H = WORLD_W` and store `self.world_h = WORLD_H`. Replace `self.box = JigsawBox.new(...)` with `self.boxes = { JigsawBox.new(5 * C.SLOT, 3 * C.SLOT) }` and add that single box to the drawer in a loop (or directly, since there's one at start) at `C.PRIORITY_PIECE`. Require `game/spawn_button` at the top of the file. Create `self.spawn_button = SpawnButton.new(WORLD_W / 2, 0, function() self:_spawn_box() end)` and add it to the drawer at `C.PRIORITY_PIECE`.

- [x] Task C ‚Äî `game/scenes/game_scene.lua` ‚Äî Add a new method `GameScene:_spawn_box()`: computes `cols = self.world_w / C.SLOT` and `rows = self.world_h / C.SLOT`; loops up to ~50 attempts picking `cx = math.random(0, cols - 1) * C.SLOT`, `cy = math.random(0, rows - 1) * C.SLOT`; rejects a candidate if it matches any existing box's `sprite.x/sprite.y` in `self.boxes` or matches the spawn button's `sprite.x/sprite.y`; on finding a free cell, creates `local box = JigsawBox.new(cx, cy)`, appends it to `self.boxes`, and adds it to `self.drawer` at `C.PRIORITY_PIECE`. If no free cell is found after all attempts, do nothing (no crash).

- [x] Task D ‚Äî `game/scenes/game_scene.lua` ‚Äî In `GameScene:update(dt)`: replace the single `if self.box then self.box:update(dt, self.pieces) end` and the single done-check block with a loop over `self.boxes` that: calls `box:update(dt, self.pieces)` on each, then iterates in reverse to find any `box.state == "done"`, sets `box.sprite.visible = false`, and removes it from `self.boxes`. Update the call to `self.player:update(...)` to pass `self.boxes` and `self.spawn_button` instead of `self.box` (new signature from Task E: `self.player:update(dt, self.pieces, self.boxes, self.spawn_button, self.drawer)`). Add `self.player.sprite.y = math.max(0, math.min(self.player.sprite.y, self.world_h - 48))` alongside the existing X clamp line.

- [x] Task E ‚Äî `game/player.lua` ‚Äî Change `Player:update`'s signature from `(dt, pieces, box, drawer)` to `(dt, pieces, boxes, button, drawer)`. Replace the existing single-box check (currently: `if self.held_piece == nil and box ~= nil and box.state == "waiting" then ... box:interact() end`) with logic that: only runs when the player didn't just pick up a piece this press, scans `boxes` (a plain array, may be empty) for the nearest entry with `state == "waiting"` within `1.5 * C.U` of the player's centre, and calls `:interact()` on that nearest one if found. Then, only if no piece was picked up and no box was interacted with this press, check distance from player centre to `button:centre()` (guard for `button == nil`) and call `button:interact()` if within `1.5 * C.U`. Leave pickup/drop/rotate logic untouched.

- [x] Task F ‚Äî `tests/test_jigsaw.lua` ‚Äî Add test coverage: (1) `SpawnButton:interact()` invokes the provided `on_press` callback exactly once per call. (2) A scene-level or extracted random-position helper (whatever `GameScene:_spawn_box()` ends up calling, or an equivalent unit-testable function) returns grid-aligned coordinates (multiples of `C.SLOT`) within `[0, world_w - SLOT]` / `[0, world_h - SLOT]`, and never returns a coordinate pair colliding with an already-occupied cell passed into it. (3) `Player:update` interacts with the nearest of several `"waiting"` boxes (not just the first in the array) when in range of more than one. (4) `Player:update` calls `button:interact()` when no piece/box is in range but the button is, and does *not* call it when a box interaction already happened on the same press.
