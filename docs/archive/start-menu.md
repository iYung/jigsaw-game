# Start Menu Checklist

- [x] Task A — `game/scenes/start_scene.lua` (new) — Implement `StartScene` per `docs/design/start-menu.md`: `Scene` subclass with `self.items = {"New Game", "Exit Game"}`, `self.selected` (1-indexed, starts at 1), an owned `lua/core/input.lua` instance mapped `{ up = {"w","up"}, down = {"s","down"}, confirm = {"e","return"} }`, and `self.manager` (the `SceneManager` passed into `.new(manager)`). `update(dt)` handles up/down wraparound selection and confirm. `_confirm()`: item 1 ("New Game") calls `self.manager:switch(GameScene.new())`; item 2 ("Exit Game") calls `love.event.quit()`. `draw()` renders a centered title plus two centered, stacked rectangles (plain solid-color fills, no art assets) with labels, selected item visually distinguished by fill color. Also implement `mousemoved(x, y)` (updates `self.selected` on hover, converting the raw window x/y into 1280×720 logical coordinates using the same scale/offset math as `main.lua`'s canvas letterboxing) and `mousepressed(x, y, button)` (button 1 over the hovered item triggers `_confirm()`).
- [x] Task B — `main.lua` — Depends on Task A. `require("game/scenes/start_scene")`; in `love.load()`, switch to `StartScene.new(manager)` instead of `GameScene.new()`. Add top-level `love.mousemoved(x, y, ...)` and `love.mousepressed(x, y, button, ...)` functions that forward to `manager.current:mousemoved(x, y)` / `manager.current:mousepressed(x, y, button)` when the current scene defines those methods (guard with `if manager.current.mousemoved then ... end` etc., since `GameScene` won't define them).
- [x] Task C — `tests/test_start_scene.lua` (new) — Depends on Task A. Cover: (1) `StartScene.new(manager)` starts with `selected == 1`; (2) pressing down wraps `1 -> 2 -> 1`; (3) pressing up from `1` wraps to `2`; (4) confirming on "New Game" (item 1) calls `manager:switch` with a `GameScene` instance — assert via a stub/spy `manager` table with a `switch` function that records what it was called with, matching the style of other test files' assertions (see `tests/test_scene.lua` for the assertion/print-PASS convention); (5) confirming on "Exit Game" (item 2) calls `love.event.quit` — the headless stubs (`lua/headless/stubs.lua`) already provide a no-op `love.event.quit`, so this just needs to not error (or spy on it if the stub supports that — check `lua/headless/stubs.lua` first).
- [x] Task D — `README.md` — Depends on Tasks A and B. Add the start menu to the Gameplay section (new game boots to a start menu with "New Game"/"Exit Game", W/S or arrows + E or mouse to navigate/confirm) and list `game/scenes/start_scene.lua` under the `game/scenes/` entry in the Structure tree, describing its role per what Tasks A/B actually implemented.
